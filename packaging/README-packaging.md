## Data file bundling

Both data loaders resolve bundled files via `sys._MEIPASS`:

| File | Bundle destination | Loader |
|------|--------------------|--------|
| `unit_converter/data/magnitudes.toml` | `unit_converter/data/` | `data_loader._default_data_dir()` |
| `unit_converter/data/Magnitudes.txt` | `unit_converter/data/` | `data_loader._default_data_dir()` (legacy fallback) |
| `Logo UC.png` | `.` (bundle root) | `resources.resource_path("Logo UC.png")` |

`data_loader._default_data_dir()` returns `Path(sys._MEIPASS) / "unit_converter" / "data"`.
`resources.resource_path(rel)` returns `os.path.join(sys._MEIPASS, rel)`.

## Application icon (Windows)

PyInstaller's `--icon` requires a `.ico` file on Windows (the repo ships `Logo UC.png`).
The build scripts run `packaging/scripts/png_to_ico.py` automatically if `Logo UC.ico` is absent.

To run it manually (requires Pillow):

```bash
pip install Pillow
python packaging/scripts/png_to_ico.py "Logo UC.png" "Logo UC.ico"
```

Once `Logo UC.ico` exists, un-comment the `icon='Logo UC.ico'` line in `UConverter.spec`
to embed the icon in the EXE.  Rebuild after any change to the spec.

## PySide6 hooks

PyInstaller 6.x ships bundled PySide6/Qt hooks that collect Qt platform plugins
(e.g. `qwindows.dll`) automatically.  No manual `--hidden-import` for Qt internals
is normally needed.  The spec includes `hiddenimports` for `PySide6.Qt{Core,Gui,Widgets}`
as a belt-and-braces guard.

If a "platform plugin not found" error occurs at runtime, add the platform plugin
directory explicitly to `binaries` in the spec:

```python
# In Analysis(...) binaries:
from PyInstaller.utils.hooks import collect_dynamic_libs
binaries = collect_dynamic_libs('PySide6')
```
